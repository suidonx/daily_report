## わかったこと
- Git Flowについて
    - Git Flowは大規模なプロジェクトに向いている。
    - ソフトウェア開発等のバージョンを管理する必要があるタイプ向き
    - その分複雑で、branchを間違えてマージすると言ったヒューマンエラーが起こる可能性がある。
- GitHub Flow
    - シンプルで開発しやすい。
    - 小規模から大規模まで使える。
    - 全フローの中でもリリースが早く、SaaSのような機能をアップし続けるプロジェクトに向いている。
    - 本番環境とテスト環境など分ける仕組みがない。

## わかったこと

- DDL, DML, TCL, DCLの分類について

- SELECT文などはDML、データ操作言語

- CREATE TABLE, DROP TABLE, ALTER TABLEなどテーブルの定義に関わるものはDDL

   - CREATEでテーブル作成、DROPでテーブル削除、ALTERで列の追加、削除

- 制約

   - テーブル作成時、各カラムに対し、データの制約をつけることができる

   - これによって異常値がテーブルに書き込まれることを防ぐ

   - NOT NULL, UNIQUE, CHECK, PRIMARY KEY, FOREIGN KEY

   - PRIMARY KEYはNOT NULLとUNIQUEを意味し、かつこの列が主キーであるという意味づけを行う。

- 参照整合性

   - 外部キーの値と、その値が参照先のテーブルで生合成を取れているか、リレーションシップが成立しているか？の状態のこと

   - 外部キーの値が、参照先のテーブルのどこにも書かれていないとき、参照性の崩壊が起きていて異常な状態。

   - これを防ぐために外部キー制約を設ける。

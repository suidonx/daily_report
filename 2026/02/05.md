## わかったこと
- コンストラクタ関数
	- テンプレートからオブジェクトを大量に作りたい時に使う
	- new演算子でコンストラクタ関数を呼び出してインスタンス化
	- クラス以外でも使える
	- Personのように頭を大文字にして関数名をつける

- プロトタイプ
	- Person.prototypeというようにアクセスする
	- Person.prototype.hello = () => {};とするとメソッドを定義できる
		- this.hello = () => {};でも定義できる
	- これは全てのインスタンスが持つ
		- Person.hello()で呼び出せる
	- インスタンス化したとき、prototypeの参照が__proto__にコピーされる
		- protoを見ることでインスタンス全てに同じメソッドを追加するメモリの無駄を省く

- コンストラクター関数の戻り値がオブジェクト
	- return {...}の時、新しいオブジェクト{...}ができる
- 戻り値がオブジェクト以外、戻り値がない時
	- this.a = a; this.b = b;の時、aとbプロパティを持ったオブジェクトが返る
	- prototypeが存在した場合、`__proto__`の中に存在
- Object.create(C.prototype)
	- C.prototypeをprototype(`__proto__`)にもつ新しいオブジェクト
	- Cのインスタンスに属する
		instanceofでtrueになる
	
- new C()の内部では
	- 空のオブジェクトを作る
	- そのprototypeにC.prototypeを設定
	- Cをthisとする
		- 関数で再現するとき、applyを使ってthisを変更する
		- 初め、thisはundefinedだが、applyで設定することにより解決
		- コンストラクタ関数を実行すると、this.a = a; this.b = b;と新しいオブジェクトにもこのプロパティが定義される。
	- オブジェクトを返す
- type ofはnullもobjectを返すので、nullではない条件を追加する

- instance instanceof C
	- どのコンストラクタ関数で生成されてインスタンスか判定する
	- instance.__proto__ === C.prototypeで確認している
		- そのコンストラクタが作った`__proto__`を持っているか見る
	- コンストラクタによってメソッドが違うので分岐させたい時使う
		- 配列とオブジェクトの分岐

- 関数コンストラクタ
	- 脆弱性の原因にもなるので普通に関数宣言して使用するのが普通

- プロトタイプチェーン
	- `__proto__`が多階層になっている
	- プロパティは階層が浅いものから探しに行く
	- in演算子はプロトタイプチェーンを遡って調べていく
	- hasOwnPropertyは自分自身に対してプロパティが定義されているか

- 継承
	- 別のコンストラクター関数を受け継ぐこと
	- プロトタイプ継承
		- ある関数のprototypeを別の関数のprototypeチェーンに追加すること

- クラス
	- コンストラクター関数をクラス表記で書けるようにしたもの
		- コンストラクター関数とprototypeのプロパティを追加するやり方のシンタックスシュガー
		- 内部ではprototypeの仕組みで動く

- スーパー
	- 継承元の関数を呼び出すためのキーワード
	- 親のコンストラクタを呼ぶ時、super()は一番最初に記述する
	- super.method()であれば、どこでも記述OK

- ビルトインオブジェクト
	- JavaScriptエンジンがコードの実行前に自動で読み込まれるオブジェクト
	- プリミティブ型以外はオブジェクト

- シンボル
	- プロパティの重複を避けるために必ず一意の値を返す関数
	- シンボルに文字列をつけることができる
	- 同じシンボル名でも内部では違う値
	- バージョンアップがあった時、既存のコードを壊さないための仕掛け
		- 一意の値を得るためのプロパティ識別子

- プロパティ
	- value以外にディスクリプターを設定できる
	- 他にwritable, enumerable, configurable, set, getがある
	- オブジェクトリテラルでは初期値true

- 演習
	- 継承は機能を拡張したい時、Userを継承したAdminUserクラスを作るといった感じ
	 - 既存の機能に影響なく、コードの機能追加しやすい設計
	 - 四則演算をするメソッドに四則演算をする関数_operatorを条件に合わせて定義する
	 - それをset(num)で数値が代入された時に計算させる

## わかったこと
- コンポーネント
	- UI の部品を表す再利用可能なコード
- デフォルトエクスポート
	- モジュールごとに1つしかexport defaultできない
	- import側で名前を好きにつけることができる
	- そのファイルのメイン関数であることを伝える感じ
- JSX要素
	- JavaScriptとHTMLタグの組み合わせ
	- 最終的にReact.createElementのような形である
	- JavaScriptの変数を使うには{}で囲む
	- JSXは「JavaScript XML」の略で、JavaScriptの中にHTMLに似た構文を書けるようにしたもの
- コンポーネントとしての関数
	- 先頭を大文字で書くのが必須
	- マークアップを返すJavaScript関数
	- 複数のJSXタグをreturnできない
		- `<>...</>`や`<div>...</div>`のような共通の親要素で囲む必要がある
- スタイル
	- CSSクラスをclassNameで定義する
	- HTMLのclass属性と同じ

- 条件付きレンダー
	- isLoggedInがtrueの時だけ右辺を表示
	- else側の分岐が不要の時、利用できる

- リストのレンダー
	- ループを使って以下のようなliタグの配列を作る
	- listitemsが配列であっても、展開してliタグ3つが入る。

- 画面の更新
	- コンポーネントに情報を記憶させたい時、stateを追加する
	- `import { useState } from 'react';`
	- コンポーネント内にstate変数を宣言できる
	- `const [count, setCount] = useState(0);`
		- useStateからは現在のstateとそれを更新するための関数が得られる

- フックの使用
	- userで始まる関数はフックという。
		- Reactの内部機能を使うための関数
		- 内部機能に引っ掛ける
	- フックはコンポーネントのトップレベルでのみ呼び出すことができる
	- 今まではクラスだけだったが、関数なのにstateを持てるのがフック
	
- props
	- コンポーネントに書かれたclassName, onClickのこと
```
<button
  className="square"
  onClick={handleClick}
>
```
- Array(9).fill(null)
	- Arrayコンストラクタで9個のemptyを作成
	- そのemptyのスロット全てにnullを埋める
- const nextSquares = squares.slice();
	- slice()でコピーの新しい配列を作る
- イミュータビリティ
	- 直接的なデータの書き換えを避けることで、結果を再利用できる
		- アクションの取り消し、やり直し
		- タイムトラベル機能
- stateのリフトアップ
	- 複数のコンポーネントで同じstateを使いたいとき、共通の親に持たせること
	- propとして子コンポーネントにデータを渡す
- Reactでインタフェースを構築するには
	- UIをコンポーネントという部品に分割
	- コンポーネントの視覚的状態を記述
	- 複数のコンポーネントを接続して、その間をデータが流れるようにする

- モダンJS
	- React, Vue, Angularなどの仮想DOMを用いるライブラリ、フレームワーク
	- npm/yarnなどのパッケージマネージャー
	- ES6以降の書き方
	- Webpack, Vite等のモジュールバンドラー
	- Babel, SWCなどのトランスパイラ

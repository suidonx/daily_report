## わかったこと
- 仮想DOM
	- 今までは、DOMを直接操作していた
		- リストの削除、追加など
		- レンダリングコストが高くなる
		- コードが複雑化する
	- これらを解決するために仮想DOMが生まれた
	- JavaScriptのオブジェクトで仮想的に作られたDOM
		- いきなりDOMを操作しない
		- JS上で仮想DOMを操作して、差分を出してからDOMに反映
		- レンダリングコストを下げる、効率化

- パッケージマネージャー
	- かつてのJS
		- 1つのJSファイルに全ての処理を記述していた
		- 複雑化、コードの再利用ができない問題
	- 他のJSファイルを読み込む
		- scriptタグを複数書くことで、複数のjsファイルを読み込む
		- コードの再利用、共通化ができるようになった。
		- しかし、読み込み順を意識しないと依存関係でエラーになる。
	- npm / yarn
		- 依存関係の解決
		- import先が明示的にわかる
		- 内部ではnode.jsが動く
	- npm / yarnの基本
		- NPM
			- パッケージのレジストリ
		- package.json
			- 設計書の役割
		- package-lock.json
			- バージョンの情報や依存関係など
			- 自動生成なので、直接編集しない
		- node_modules
			- 各モジュールの実態が入っている
			- jsファイル本体
			- 膨大なのでGitHubにはあげない

## わかったこと
- モジュールバンドラー
	- 複数のjs/css/imageファイルを1つにまとめるためのもの
	- 本番環境で実行する時、読み込みファイルが多いとパフォーマンス低下
		- 本番用にビルドする時、1つのファイルにまとめたい
		- 依存関係もよしなにやりたい。
	- webpackやVite
- トランスパイラ
	- 新しいJavaScriptの記法を古い記法に変換する
	- ES6の記法は、対応していないブラウザ(IEなど)がある。
		- BABEL, SWC
- SPA
	- モダンJSはSPAが基本
	- HTMLは1つで、JavaScriptで画面遷移をする
	- サーバーから返すのはhtmlではなく、データを返す
	- メリット
		- ページ遷移ごとのチラつきがなくなる
		- 仮想DOMを使うので、表示速度がアップ、UX向上
- 分割代入とデフォルト値
	- `const {age, name = "guest"} = myProfile;`
	- 分割代入しつつnameが定義されていないときはデフォルト値が入るようにする
- &&と||
	- 短絡評価で&&の場合は左辺がfalseなら、そこで評価をやめて左辺が評価結果となる
	- 左辺がtrueなら右辺
	- ||の場合、左辺がtrueなら短絡評価で左辺が結果となる
	- 左辺がfalseなら右辺まで見にいくので右辺が評価結果となる。
- deleteButtonのイベントリスナー
	- deleteButtonを起点としてcloset()を使ってliタグを見つける
- li.remove()で良さそう
	- 親のDOM要素を取得してremoveChild()という書き方が昔からある記法
	- これは親が子を管理するというDOMの思想によるもの
	- ただ今はremove()という便利な書き方がある
- DOMの実体への参照
	- li.innerText="hoge"などで中身を変更できることからもわかる
- appendChildをすると親子の関係が生まれる。
	- 同時にappendChildすると、すでに親子の関係がある場合は、その親との縁が切れて、新しい親子の関係が作られる、結果：移動したように見える
- vanila JSではシンプルなTodoアプリでもコードが複雑で、保守が難しい
- コンポーネント志向
	- 関心の分離で、色々な場面で再利用できるコードができる
	- 振る舞いの部分と、見た目の部分で分けるなどなど。
- createRootがindex.htmlに使われている
	- React v18の書き方
	- DOMの\#rootを取得して、createRoot()の引数に渡すことで、Reactの管理領域であることを宣言する
	- 仮想DOMで管理することを宣言
- root.render()
	- Reactコンポーネントを描画する
- StrictMode
	- 開発中にバグを見つけやすくするチェックモード
	- 危険な書き方を警告してくれる
- React.Fragment
	- <>が省略記法
	- 複数のHTMLタグをreturnするときにdivタグ等で一つにまとめる必要がある
	- divタグ等を使いたくないときにこれをつかう
- コンポーネント
	- ファイルはjsx拡張子
	- 大文字はじまりのパスカルケース
- CSSの設定
	- JSのオブジェクトにプロパティと値を記述する
	- font-sizeではなくfontSizeのキャメルケースになる
	- 値は文字列と数値で指定していく
- props
	- コンポーネントに渡す引数のこと
		- オブジェクトとしてコンポーネント側で受け取る
	- 動的にコンポーネントの見た目を変更する
- state
	- コンポーネントが持つ状態
	- useStateというフックを使う
		- フックを使う関数はコンポーネントの最上部でしか使えない
		- 配列を返すので分割代入[]で受け取る
	- setNumが呼ばれた瞬間にDOMが操作されるわけではない。
	- stateが更新されたら、App.jsを再度読み込み、レンダーする。
		- いちいちsetNumが呼ばれた瞬間にレンダーとすると無駄
		- 同じイベント内の更新はまとめて、最後に1回レンダーする（コンポーネント関数を読み込む）
		- numがレンダー時点のスナップショットとなる
	- 関数の引数には最新のstateの値を返す
```
const onClickCountUp = () => {

	setNum(num + 1); // 0 + 1
	setNum(num + 1); // 0 + 1
	// パフォーマンス向上のため、バッチ処理になり、numの更新は後回し
};
```

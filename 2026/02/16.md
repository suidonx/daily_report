## わかったこと
- TypeScript
	- 型を指定することで、暗黙的な型変換を防ぐ。
		- "1000"を1000と変換してしまう。
	- どの型が引数に入るか明示して、わかりやすくする。
	- コードを書くときに、引数に入る変数の型が間違っていると、エラーを表示してくれて開発効率が上がる
```
const getTotalFee = (num: number): number => {
	const total: number = num * 1.1;
	return total;
};
```
- typeを使って型定義をする
```
type TodoType = {
  userId: number;
  id: number;
  title: string;
  completed: boolean;
};
```
- 型を明示すると、補完機能が効くようになるのが嬉しい
- 存在しないプロパティにアクセスしようとしていると警告を表示してくれる
- コンポーネントのpropsに型を定義する
	- propsに存在しないプロパティにアクセスしようとすると警告が出る
	- propsに渡していないプロパティがある時も警告が出る
		- Pickを使うと不要なプロパティがコンポーネントに渡されなくとも、警告を出さないようにする
		- Omitを使うと、型の中で、一部無視するプロパティを決められる
			- Pickの余事象ver
- 型定義を共通化する
	- 別ファイルに切り出す
- 関数コンポーネントの型定義
	- FC(Functional Component)を使う
		- React.FC\<Props>
	- React 18で暗黙的なchildrenが削除された
	- 使う場合は明示する
	- https://zenn.dev/motonosuke/articles/13d0fdd417ec7c#%E7%B5%90%E8%AB%96
	- あまり使わないっぽい
- プロパティのオプション（任意）
```
type Props = {
  name: string;
  age?: number;
};
```
- ライブラリの型定義
	- 型チェック、補完、エラー検出のために役立つ
	- `@types`が接頭辞につく
```
// sum.d.ts
export function sum(a: number, b: number): number;
```
- カスタムフック
	- コンポーネントからロジックを切り出してカスタムフックとして運用
	- フック名はuseで始める
- React環境構築
	- npm（Node Package Manager）は、JavaScriptのライブラリやツールを管理する仕組み
	- Node.js をインストールすると一緒に使えるようになります。
	- React 本体や関連ライブラリは、すべて npm を通して導入します。
- ビルド

		- モジュール変換

- Chakra UI
	- UIコンポーネントライブラリ
	- コンポーネントに一貫性を持たせたUIを楽に実装できる
	- ユーティリティファースト
	- Tailwindと比べると、デザインに制限があるので自由に作りたい場合はTailwind

- Providerコンポーネントはルートのコンポーネントに対して使う
	- チュートリアルの通り

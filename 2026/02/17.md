## わかったこと
- React Router v6
	- HeaderLayoutを当てるとき
		- 親コンポーネントのelementにHeaderLayoutを指定する
		- HeaderLayout内でOutletコンポーネントで子コンポーネントを表示していく
- React Router v5
```
type Props = {
  children: ReactNode;
};

export const HeaderLayout = ({ children }: Props) => {
  return (
    <>
      <Header />
      {children}
    </>
  );
};

```
```
<HeaderLayout>
  <Home />
</HeaderLayout>

```
- HeaderLayoutのChildrenはHome
- childrenをReactNodeと型定義しないとTypeScriptではエラー表示が出る
- {children}の部分がv6ではOutletコンポーネントで表示。
	- OutletはHeaderLayoutを渡してる親コンポーネントの子コンポーネントが入る
- まとめ
	- 親のelementにHeaderLayout指定すると子にそのレイアウトを指定できる
		- HeaderLayoutを囲む必要なし
		- HeaderLayout側で子コンポーネントをOutletで表示させる
- 嬉しさ
	- 共通UIの実装のしやすさ
	- HeaderLayoutを親に1回書けばいい

- flexGrow
	- 余ったスペースをどれくらい割り当てるか
	- 1つの要素だけにflexGrowを指定すると、その要素が余った余白を埋めるように広がる
	- 2つの要素に指定されており、それぞれのvalueが1, 2だったら余った余白を1:2で分配する

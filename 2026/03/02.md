- 型の区別
	- 名前的型付け
		- Java, PHP, C#
		- 型の名前が違うと互換性はない
		- Person型にDog型のインスタンスを代入できない
			- 互換性がないから
	- 構造的型付け
		- 型が持つプロパティやメソッドの構造が同一であれば、異なる型同士でも互換性があると判定
- 部分型
	- 型と型の関係を階層関係で捉えることができる
	- 頂点に行けば行くほど抽象的で、階層を下に進むほど具体的
	- 上位にいる型を基本型
	- 下位にいる型を部分型
	- 部分型は基本型と互換性があるので、基本型に部分型の値を代入できる。
		- CircleとRectangleは異なる型だが、同じShapeとして扱える
	- 名前的部分型のJavaなどではextendsによって基本型と部分型の関係があることを示す
		- 基本型が親、部分型が子
	- 構造的部分型
		- 型が持つメソッドやプロパティの構造に着目して、基本と部分の関係性を判断する
		- 構造が似ている（同じメソッド名のメソッド、プロパティを定義している)
		- extendsでは部分型かどうか判定する基準にはならない
```
type Summary = { name: string };
type Detail = { name: string; age: number };
 
const johnDetail: Detail = { name: "John", age: 28 };
const summary: Summary = johnDetail; // 代入できる。構造的部分型として互換があるため
 
const johnSummary: Summary = { name: "John" };
const detail: Detail = johnSummary; // 代入できない。構造的部分型として互換がない（ageを含まないため）
```

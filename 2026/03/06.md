### Next.js
- Reactとの比較
- 空のHTMLを生成して基本JSXで記述する
```
<body>
    <div id="root"></div>
    <script type="module" src="/src/main.tsx"></script>
  </body>
```
- JSXで記述したものはトランスパイルが必要で、全てブラウザでHTMLを構築
	- 不必要な部分までJavaScriptでHTMLを構築
		- デメリットとして
		- 初期表示が遅い
		- SEOに弱い
- Next.jsでは、サーバーでレンダリングできるようになった。
	- レンダリングとはJavaScriptでHTML生成
- Reactではサーバーにあるのは上記のようなdivタグのHTMLファイルのみで、JSファイルでHTMLを生成
- サーバー側でJavaScriptを実行して、完全なHTMLファイルを提供するようになった。
	- 上記の問題点を解消
- Next.jsはReactベースのフルスタックフレームワーク
	- 以下の機能が最初から使える
	- Routing
	- SSR,
	- SSG
	- API Routes
	- Server Components
	- Build
- ReactはUIライブラリという位置付けで状態管理、コンポーネント、仮想DOM更新などができる
	- RoutingはReact Routerで、ビルドはViteを使うなど無い機能は組み合わせる必要がある
- Djangoと違いORM、DBMSがない
	- ORMはPrisma
	- DBMSはPostgreSQL、SQLiteなどなど利用していく

- 静的レンダリング
	- データの取得とレンダリングははビルド時またはデータのrevalidate時にサーバー上で行われる
	- ユーザーがアクセスするたび、キャッシュされた結果が提供される
	- 利点
		- 高速でキャッシュを配信
		- サーバー負荷の軽減
			- コンテンツはキャッシュされるため、ユーザーのリクエストをもとにコンテンツを生成する必要がない
		- SEO
			- 事前レンダリングされたコンテンツはページが読み込まれた時点でコンテンツが利用可能であるため、検索エンジンのクローラーによるインデックス作成が容易
- 動的レンダリング
	- コンテンツはリクエスト時にレンダリングされる
	- 利点
		- リアルタイムデータ
			- データが頻繁に更新されるコンテンツ
		- ユーザー固有コンテンツ
			- ユーザーの操作に応じてデータを更新
		- リクエスト時のみ知ることができる情報にアクセス
			- CookieやURL検索パラメータ
	- 課題
		- データ取得中はページ全体がブロックされ、UI表示ができない
		- アプリケーションの速度は最も遅いデータ取得と同じ
- ストリーミング
	- ページを小さなチャンクに分けて、データが準備できたらサーバーからクライアントに送信する技術
		- ユーザーはページ全体を待たずに操作可能
		- 従来は全データが揃うまでページは表示されなかった
	- 各コンポーネントをチャンクとみなす
- ページ全体をストリーミングする
	- loading.tsx
		- React Suspenseをベースに構築されたNext.jsファイル
		- ページコンテンツの読み込み中に表示されるフォールバックUIを作成できる。page.tsxのコンテンツが揃うまで表示。
		- 静的コンテンツはすぐに表示されるし、操作も可能。
			- 割り込み可能なナビゲーション
- ルートグループ
	- ()で囲んだフォルダを作るとURLには現れずフォルダ構造だけ整理できる
	- /dashboard/loading.tsxに置くと、/dashboard配下の全てのページに適用されるので、/dashboard/invoices/page.tsxも影響を受ける
- コンポーネントのストリーミング
	- Suspenseを使って、細かく特定のコンポーネントをストリーミングする
	- ボトルネックとなる遅いデータリクエストだけをストリーミングするさせて、ページの残りUIをすぐに表示させる、こともできる
	- 一般的には、データ取得を必要とするコンポーネントをサスペンスでラップするのが良い方法
		- データ取得はそのコンポーネントに記述
	
### Next.js完全攻略
- App Router(version 13以降)
	- appディレクトリをルートに置く
	- ページごとにレンダリング戦略を選択できる
		- App Route
			- Static Rendering
			- Dynamic Rendering
- Static Rendering（SSG: Static Site Generation) 
	- ビルド時にレンダリングしてHTMLを生成
	- 表示が早い
	- CDNキャッシュ可能
- Dynamic Rendering(SSR)
	- リクエストごとにサーバーでレンダリング
	- ユーザーデータ、リアルタイムデータ、認証などで必要
 - ISR (Incremental Static Regeneration)
	 - SSG
	 - 指定した間隔でHTMLを構築
- ハイドレーション
	- サーバーで作ったHTMLに後からJSを追加
- appディレクトリ配下に置かれたファイル
	- ファイル名によって特別な役割を持つ
	- page.tsx
		- 配置したディレクトリによってURLパスが自動生成
		- 対応するURLパスに表示されるページがpage.tsx
- クライアントでデータ取得の際、直接データベースにクエリを発行しないこと
	- DBにアクセスするための情報がクライアント側に残るから
		- host, user, password
	- サーバーを間に挟むのが正しい
		- API経由するか
		- Server ComponentからDBにアクセスする
			- サーバーでJS実行するのでAPIレイヤーが不要
			- Databaseから直接データ取得
- エイリアス
- 相対パスだと../../..のようになってみにくい
- エイリアスをつけるとプロジェクトルートを基準としたパスを書ける
	- 厳密には絶対パスではなく、仮想的な絶対パス
```
    "baseUrl": ".",
    "paths": {
      "@/*": ["./*"]
    }
  },
```
- React Server Component(RSC)
	- React v18に追加された機能
	- サーバーでコンポーネントをレンダリングする技術
- Client Component
	- 必要に応じてクライアントでレンダリング
		- useEffect, useState
		- ブラウザAPI、イベントハンドラーなど使える
	- いつものReactのような形で書ける
- Server Component
	- Dynamic API, Prismaなどサーバーで動くものが使える
	- 非同期コンポーネントとすることができ、クライアントコンポーネントよりもシンプルに書ける
- ページはデフォルトでStatic Renderingして、Dynamic APIを利用しているとDynamic Renderingとなる
	- 利用していなくとも、fetch関数にcacheオプションでDynamic Renderingにできる

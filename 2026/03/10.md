## わかったこと
### Next.js 完全攻略
- Parallel Route
	- 同じレイアウト内で複数のページを同時に表示する機能
	- 例えば、@header/default.tsxをapp配下に作ると、同じセクション内のlayoutでpropsにheaderを受け取ることができる
		- DashborardとHeaderを同じレイアウト内で表示する
	- 以下のInterceptと組み合わせてモーダルをアクセスするときに使える
	- photo一覧を背景にしながら、photo詳細をモーダルで出すなど。
- Intercepting Route
	- 別のルートをインターセプトできる機能
	- Parallel Routeと組み合わせる
	- (.)同じ階層
	- (..)1つ上
	- (..)(..)2つ上
		- (.)usersの時、/usersにアクセスすると、同階層の/usersのpageを横取りして(.)users/page.tsxが表示される。リロードすると、本来の/usersが表示される。
- フロントエンドとバックエンドの統合
	- ORMライブラリ
		- バックエンドの開発
			- データベースCRUD操作
	- Route Handlers
		- APIの作成
	- Server Component
		- サーバー内でレンダリングするので、サーバー内でORMツールを呼び出せる
	- Server Actions
		- サーバーで動く関数
	- フロントエンドと近い距離でバックエンド開発ができる
- CRUD
	- Read
	- データ取得
		- Route Handlers
			- サーバー側でAPIを作成してClient Componentでデータ取得して表示する
				- 手間がかかる
				- HTTP通信が必要になる
				- JSバンドルが増える→ブラウザでレンダリングが必要
		- Server Component
			- サーバー側で直接ORMを操作してデータ取得可能、API作成が不要
			- サーバーでレンダリングするので、ブラウザ側のパフォーマンス向上が見込める
	- Create, Update, Delete
		- Route Handlers
			- データ取得と同じ
			- APIを作成して、Client ComponentでPOST通信を送る
		- Server Actions
			- サーバーコンポーネントで呼び出すか、クライアントコンポーネントで呼び出すか
			- Client Componentの方がUXが良い
				- formをstateで持たせることで、バリデーションがリアルタイムにできるから。
- Server Actions
	- use serverと宣言するとServer Actionになる
	- 基本Server Componentだが、Client ComponentでServer Actionを関数として呼び出すことができる 
	- クライアントコンポーネントだけでなくサーバーコンポーネントでもバリデーションを記述する
		- クライアントコンポーネントのバリデーションは即座に行われ、ユーザー体験がよくなる
- useActionState
	- エラーハンドリング
	- フォーム送信中のstate管理を少ないコード量で書けるようになる
- Link
	- アプリケーション内のページのナビゲーション
	- aタグと異なりJavaScriptでページ遷移するため、フルリロードしない
		- プリフェッチで実現
			- Dynamic Renderingのページはオプションをつければできる
- Image
	- imgタグの拡張コンポーネント
	- パフォーマンスとユーザー体験の向上
	- next.jsで画像配信のエンドポイントを立てて、public/の画像を配信
	- webPフォーマットにしてファイルサイズ少なめで高品質の画像を配信
	- ファイルサイズやクオリティを加工して読み込み速度の向上
	- 画像の遅延読み込み
		- スクロールして画像のある場所に近づいたときにダウンロードするようにしている
- useRooter
	- replace
		- 現在のページを置き換える
			- 戻るを押しても、replaceを押す前のページではなく、その前のページになる
			- 現在のページを置き換えているから
		- 履歴を残さないために使える
	- refresh
		- Server Componentの再レンダリング

- usePathname
	- 現在のパスを取得
- useSearchParams
	- クエリパラメーターを取得する
	- getでキーを指定すると値を取得
- redirect
	- useRouterのpushは現在、表示部分の差分を受け取る
	- 完全なHTMLをサーバーから受け取る
	- 認証やエラー処理の自動的な制御の際にredirectを使用
	- ユーザーのアクションの際はuseRouterのpush
- notFound()
	- not-found.tsxを開く
	- データベースからデータを取得して、存在しなかった場合、表示するのがよい
- revalidatePath/revalidateTag
	- Next.jsでキャッシュされたページ、データを再検証するための関数
		- キャッシュ削除して再レンダリング
	- Static Renderingされているページでデータを更新したいときに使える

### Prisma完全攻略
- ORM
	- Object Relational Mapping
	- オブジェクトを操作することでDBの操作ができるツール
- Prismaの特徴
	- 自動マイグレーション
	- JS/TSのコードでクエリを表せる
	- 型安全
		- コードの自動補完、ミスに気付ける
- npm audit fix
	- 脆弱性のあるパッケージを安全なバージョンにアップデート
- prisma generate
	- schemaに基づきprisma cilentを生成
		- 特に指定しないとnode_modules/@prisma/clientに作られる
	- `npx prisma init --db --output ../app/generated/prisma`
		- 初期化
			- schema.prismaを作成
			- .env作成
		- dbオプションでschemaにDBの接続先情報など記述
		- outputでPrisma Clientの生成場所を指定している
		- schemaに書き込む
- prisma migrate dev
	- schemaに基づいて、DBに反映させる
	- prisma generateも含んでいる
	- `npx prisma migrate dev --name init`
		- マイグレーションの名前をnameオプションで指定
		- 省くとマイグレーション名をターミナルで入力
	- --create-only
		- マイグレーションファイルのみ作成
		- DBに反映しない
- prisma migrate deploy
	- 本番環境用
- schema.prisma
	- datasource, generator, modelで構成
	- datasource
		- database_urlはprisma.configに記述するようになった
	- generator
		- outputの場所にPrisma Clientを生成することが多くなった
	- model
		- テーブル構造
		- カラム名と型、属性を記述
			- 型にモデル名がある場合は仮想的なカラム、DBのフィールドには存在しない
		- 属性
			- 追加情報、振る舞いを追加
			- @default, @id, @@unique
			- 文字数制限
				- Stringのtype mappingを変更する

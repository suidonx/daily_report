## わかったこと
- server actionはclient componentでもserver componentでも受け取れる
	- client component
		- 戻り値を受け取れる
		- UX向上
	- server component
		- パフォーマンスが良い
	- という使い分け
- server actionをformタグに渡す
	- formDataのnameをキーとして、inputタグで送信したvalueが格納される
	- `formData.get("hoge")`でvalueを受け取れる
		- formDataをオブジェクト化として受け取る
- react-hook-form
	- zodと連携してformのバリデーションを楽に書ける
	- 入力をstateで管理
		- 通常はonChangeにより1文字ごとに再レンダリングされてしまうが、不要なレンダリングを防いでくれる
- next-safe-action
	- if文で実装していたバリデーションをzodと連携して、コード量が少なく実装できる
	- バリデーションが通ったらactionに渡した関数を実行する
- prismaのCRUD操作
	- エラーハンドリングのためtry-catchが必要
- zod-prisma-types
	- prismaのschemaからzodのschemaを自動生成する
	- OptionalSchema
		- default値がついたフィールドをoptionalとするschema
- `///`
	- prismaのドキュメントコメント
	- zod-prisma-typesと連携してエラーメッセージのカスタマイズができる
	- https://www.npmjs.com/package/zod-prisma-types
- regex(/[a-zA-Z]/)
	- 文字列の中に英字が含まれているかチェック
	- 文字列を1文字ずつチェックして、一つでも当てはまればOK
- shadcnのreact-hook-formを使ったformUI
```
render={({ field, fieldState }) => (
            <Field data-invalid={fieldState.invalid}>
              <FieldLabel htmlFor="form-rhf-demo-title">Bug Title</FieldLabel>

```
- renderに渡すfieldはフィールドオブジェクト
	- nameに応じたvalue, onChangeを持つ
	- Inputに渡すことでnameの値を表示する
- next-safe-actionとreact-hook-form
	- useHookFormAction
		- フロントでZodバリデーション

- zodエラーマップ
	- エラーによってどのメッセージを出すか、マッピングしたもの
	- エラーメッセージをcodeによって条件分岐させて、メッセージをカスタマイズ
	- issue.codeで条件分岐
- zod エラーメッセージをカスタマイズするには
	- `zod.string(),min(1, {message: "hoge"}))`のように直接渡す
	- zodエラーマップでエラーコードごとにカスタマイズ
	- z.config({customError})でエラーマップを指定
	- https://zod.dev/error-customization?id=global-error-customization
		- 一応、簡単に日本語化できる
		- https://zenn.dev/y_megane/articles/20250524-zod-v4-localization
	- ZodProviderを作成して、childrenにエラーマップを渡す
		- layoutに配置する
		- z.configを適用させる
		- エラーをリアルタイムに表示させたいのでClient Componentでないと動かない
		- Compositionパターン
			- Client Componentの中のchildrenにserver componentを渡すことでclient -> serverができる
- 副作用import
	- `import "@/lib/zod"`
	- 設定ファイルの実行だけさせたい時。
- prisma findUnique
	- unique制約のカラムで1件取得
- bcryptjsを使ってパスワードのハッシュ化
- react-hook-formのfile input
	- inputタグに{...field}と渡すとe.target.valueを読もうとしてエラーになる
	- 渡さずにonChangeにてfileを渡す
- `.refine((file) => file.type.startsWith("image/"))`
	- file.typeでMIMEタイプを見る
		- image/png, image/jpegなど
